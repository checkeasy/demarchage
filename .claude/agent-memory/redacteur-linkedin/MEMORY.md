# Memory Redacteur LinkedIn - CheckEasy

## Context Produit Sauvegardé
- **Produit** : CheckEasy (SaaS pour conciergeries et location courte durée)
- **Audience** : Property managers, conciergeries, gestionnaires locatifs (100% France)
- **Redacteur** : Adrien
- **Ton** : Décontracté, direct, humain (pas de jargon marketing)

## Patterns Confirmés
- Demandes de connexion : MAX 300 caractères (jamais de pitch produit)
- Messages de follow-up : MAX 500 caractères
- Signature : Toujours "Adrien" (pas de titre pompeux)
- Personnalisation : {{prenom}}, {{entreprise}}, {{ville}} (obligatoire)
- Pain points CheckEasy : inspections manuelles, documentations, coordination arrivées-départs

## 5 Segments LinkedIn Mapped
1. **Leads Chauds** : 3 touchpoints (J0, J+1, J+4)
2. **Perdus** : 2 touchpoints (J0, J+2) — Email en priorité
3. **À Recontacter** : 2 touchpoints (J0, J+3)
4. **Jamais Contactés** : 3 touchpoints (J0, J+1, J+5)
5. **Contactés Sans Réponse** : 2 touchpoints (J0, J+2) — Email en priorité

## Multi-Canal Guidelines
- LinkedIn toujours = warm-up ou relance
- Email = reconnexion ou "nouvelle approche"
- Délai min entre touchpoints same-channel : 2-3 jours
- Référence cross-canal : naturelle, jamais forcée ("on a échangé sur LinkedIn...")
- Jamais relancer 2x même canal sans 4+ jours d'écart
